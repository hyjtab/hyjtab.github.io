深入理解计算机系统：一至三章读书笔记

Chap1.漫游

本章实际上没有什么内容，只是粗浅的谈及了CPU的硬件组成，首先指出了计算机的主要处理对象——信息的本质，位+上下文，一个c语言编写的helloworld程序文件，与他编译生成的可执行文件所包含的信息是不同的，区别不仅仅是某个具体的位不一样，还有上下文不同，两个因素导致了他们的执行效果也不同。
然后介绍编译细节，首先是编译器进行预处理和转换语言的工作（生成.i与.s文件），然后是汇编器翻译汇编码为二进制文件（生成.o文件）最后由链接器生成可执行文件（PE或ELF）
本章还介绍了本过程数据在硬件中的大致流动情况，在那之前稍微的介绍了一些有关计算机组成相关的知识：CPU是人人都知道的，伸出的I/O总线连接着计算机的各个I/O设备，总线的条数决定了机器的“字长”，最重要的DRAM（主存）是在执行程序时，用来存放程序和处理程序的部分，后面会提到的栈也在这里存放。接着就是CPU的细节了，分为总线接口，PC（程序计数器），ALU（算术逻辑单元），寄存器文件，在任何开机的时刻，PC都指向主存中的某条机器语言指令。ALU与寄存器文件共同执行一些简单的操作，执行完成后更新PC，而所谓寄存器文件就是一个小的存储设备，由一些单个字长的寄存器组成，每个寄存器都有唯一的名字，程序员通过名字访问他们。
流动情况是：首先当我们运行helloworld程序时，提供给shell一个指令，之后shell程序执行一系列指令（启动了字符对应的函数功能）加载可执行的hello文件，将hello文件从磁盘复制到主存，然后将控制权转交给hello文件中的函数，执行你预设的命令。值得一提的是，hello程序加载到主存的过程，是占用了I/O总线的DMA（内存直写通道技术）技术实现的。
然后讲述了高速缓存的概念：容量越大的存储系统，读写开销就越大，读写开销越小的存储系统，就相对昂贵，容量也不会很大。因此一般的计算机都会有高速缓存存储器（cache）,其中较低层的L1,L2缓存用一种叫做（SRAM）（静态随机访问存储器）来实现,L3则用（DRAM）（动态随机访问存储器）实现。值得一提的是，固态硬盘与机械磁盘所用的存储方法也不同，而且更大，更廉价，更慢。
然后注意到一个细节，shell程序究竟如何启动了hello程序呢？谁提供给了他hello程序的相关信息，并指示软硬件一起配合hello的执行呢？那当然就是操作系统了。此处提到了操作系统的两个基本功能，首先保证硬件不被失控的程序滥用。然后为应用程序提供更方便使用的操作接口与机制。进行了如下几个比较重要的抽象：进程（处理器，主存，I/O设备的抽象），虚拟内存（主存与磁盘设备的抽象），文件（I/O设备的抽象）。
然后开始介绍进程，线程，虚拟内存，文件，这几个抽象概念。进程会为每个程序提供一种只有我在运行的假象。是操作系统对一个正在运行的程序的一种抽象。而在进程中可同时运行多个线程：每个线程都运行在进程的上下文中，并共享同样的代码和全局数据。（网络编程并发是一种多线程）
然后介绍虚拟内存的概念他为每个进程提供了一个不同于物理地址的空间块，并将其分为5段左右：程序代码和数据段，堆段，共享库段，栈段，内核虚拟内存段。
而文件就是字节序列。LINUX中，所有东西都可以被抽象成为文件——它为应用程序提供了一个统一的视图——而同一个程序可以在使用不同磁盘技术的不同系统上运行。
随后预告了网络服务器编程及使用。
在了解了如此多的新内容后，又介绍了系统优化的Amdahl定律——要想显著加速整个系统，必须提升全系统中相当大的部分的速度。
然后介绍了并行与并发，预告了之后我们将如何使用并行并发来加速代码运行过程。举例有：
线程级并发：首先一般并发就是上下文切换，以一种人意识不到的状态实现。然后是多核并发，实现了真真正正的同时操作。超线程则使利用一个线程的垃圾时间，执行另外一个线程的技术，让4核CPU同时执行8个及以上的线程。
指令集并发：主要介绍了之前的通过流水线来实现的CPU模型。然而现代处理器用乱序执行模型能够达到更加可怕的并发执行效果。
单指令，多数据并行：SIMD并行。一条指令同时对多个数据进行操作。浮点数向量是典型代表。
最后再次强调了系统抽象的重要性，并点出：虚拟机是更高级别的对整个PC的抽象。
第一章就此结束。

Chap2.信息的表示与存储

本章讲述了存储系统是如何通过01代码，将计算机内的各种数据，字符表示出来的。介绍了无符号数，补码以及浮点数的表现和运算机制。对于溢出现象的本质也进行了发明。
开篇讲述了编译时的命令区分：

gcc -std=gnu89||c89||c99||c11
gcc -ansi                       #几乎等于c89

进入正文，首先明确了地址这一概念是服务于字节单元的，每个字节8个bit，指针是近似于地址的概念。然后介绍了老生常谈的进制与进制转换问题。随后嵌入介绍了字长的概念——它指明了指针数据的标称大小——虚拟地址就是这样的格式进行编码的——字长决定了虚拟地址的大小——即主存空间的抽象大小。
64位机器兼容32位机器，是一种向后兼容。但更低位的机器则不行了。在不同字长的机器上，各种数据类型的长度是不同的。具体可在网上查询。而这就造成了程序员对可移植性的考虑，因此数据类型的范围也应该加入考虑范围内。
接下来讲到寻址规则，一个对象的地址是归属于他的第一个字节。在内存中的排列则有两种：“小端序”“大端序”，小端序高字节放在高地址处，大端序相反。大多数Intel兼容机都采用小端序模式。而IBM或其他小型机器采用大端序。但并不是只有微处理器单一的决定了数据的大小端问题。操作系统也是这个问题上的主角。基本所有的操作系统都默认为小端序。大小端在网络交流，反编译的过程中起作用。
然后书中提供了一个重要程序，show_bytes。

void show_bytes(char * start,unsigned long len){
    unsigned long i;
    for (i = 0;i < len; i++) {
        printf("%.2x", start[i]);
    }
    printf("\n");
}

通过这个函数可以查看大端小端状况。随后介绍了布尔运算，逻辑表达式，移位运算，其中的逻辑右移进行了重点的介绍。
进入正题，介绍了整数的表示。首先是无符号数：就只是简单的二进制源码——无符号数与二进制编码的关系是一个双射。然后介绍了补码。补码的定义是最高位取负，其余位表示为正。这样做可以发现原本0-15的范围变成了-8-7的范围……对每一个编码，都有唯一的有符号整数相对应。我们注意到，由于0这一自反元的存在，导致最大的有符号整数与最小的有符号整数相加不为0。另外根据反码的数学性质，得到，补码就是各位取反加一得到的结果。
随后介绍了有符号与无符号数的转换。对于正数，相等。而负数，则为负数+此类型无符号数最大数+1所得。反之同理。
然后介绍了扩展与截断，截断操作是一种取模运算。然后介绍了有、无符号数转换可能造成的无限循环问题。
然后进行到了加减乘除运算。
加减运算，
对于无符号数来说，若发生了溢出，则结果为在正确结果的基础上，减去最大可表范围数得到的值。检测溢出的方式：s = x + y，若s < x或s < y；则发生了溢出。
对于有符号数来说，若发生了溢出，有两种情况，正溢出同无符号数，负溢出则在正确结果的基础上，加上最大可表范围所得的数。检测溢出则由得数符号来进行判断。
乘除运算，
对于无符号数来说，默认进行了取模操作。有符号数则先转换为无符号数再进行取模。
乘、除以二的幂——逻辑左、右移位。

终于我们来到了浮点数。
浮点数的表示有许多标准，我们介绍了IEEE标准：由符号位（1bit），阶码（8bit，11bit），尾数（23bit，52bit）表示的浮点数。
阶码字段的值决定了此浮点数的类型：当阶码字段不全为零或全为一时，浮点数为规格化值，当全为零时，表示非规格化值。当阶码全为一时，表示无穷大或无穷小，另外一类表示“不是一个数”。
对于规格化值，阶码的值不等于e的值，而是应当减去一个偏置量bias，让他在正负数左右平均分配。一般来讲，范围相当于有符号数的最大值到有符号数的最小值加二。而顺序上0000…1为最小值——没有固定的位推算值公式！而尾数表示小数字段。注意，每个小数字段前都会加一，尾数表示的值的每个值都在范围1-2之间。
对于非规格化值，正零和负零只允许符号位为1（负零）其余位为零。当阶码字段全为零时。阶码E的值为1-bias，也就是此序列表示成有符号数的最大正整数。尾数的值为尾数的值。
当阶码全为一时，其余位全为零，表示为正无穷，符号位为一时，表示正无穷。其余情况为NAN（NOT A NUMBER）
然后介绍了舍入这一概念——由列表可知，浮点数的排布在数轴上不是均匀的。越靠近零数字越多，越密集，但总有表示不了的浮点数，这就需要进行舍入。舍入中向偶数舍入，一方面尊重四舍六入，另一方面在不能进行四舍六入的方法时，向偶数方向舍入，使得数值更加科学。
随后介绍了浮点数运算的几个性质：不能使用结合律、转换时有安全风险、时刻伴随舍入。

chap3.程序的机器级表示

本章全部内容为介绍汇编语言。
首先介绍了处理器的发展史。然后开始介绍一些基本操作指令。这里介绍一下PUSH和POP这两个指令：
PUSH：R[%RSP]<-R[%RSP]-8;M[R [%RSP]]<-S;先减，再入。而POP操作则是先出，再加。然后指出了LEA操作这一简算操作命令。随后介绍了IMUQ与IDIVQ——这两个操作使用了128位的数型。
之后介绍了控制转移指令：跳转指令，其编码方式为PC相对的，也就是他们会将目标指令的地址与紧跟在跳转指令后那条指令的地址之间的差作为编码，可以为1字节，2字节，4字节。也有绝对编码，用四个字节制定目标。这些编码由汇编器与链接器有关。
然后介绍ifelse的逆向工程。特别指出了用条件传送来实现条件分支。执行的翻译类似于下文：

if(a>b){
    return a-b;
}
else{
    return b-a;
}

变成了：

x=a-b;
y=b-a;
if(b>a)x=y;
return a;

其中后面的if语句用了条件传送语句实现。但是，条件传送并不总是安全且高效的，它一方面会导致本不应发生的错误——执行不应执行的语句，另一方面如果两个运算都特别冗长，可能结果不如优化之前的快。
然后介绍了循环语句的逆向工程。其中while可用两种方式翻译：jump_to_middile,所有判断都在循环块内执行，有一个无条件跳转指令打头阵。还有一种方法就是grounded_do——把while循环翻译成了dowhile，先设一有条件循环，如果满足该条件，就跳出，如果不，则进入dowhile循环，o1优化都这样做。而for循环自然也就水到渠成。
之后介绍了跳转表。跳转表的具体分析，首先根据反汇编得到的钱买你的条件转移语句，得到default的标号，然后根据rodata部分的表头，得到剩余标号的分布情况，其中标号排列顺序为0~最大的顺序。
之后介绍了运行时栈。过程的实现离不开以下机制：传递控制，传递数据，分配和释放内存。
关于这个重要概念，可能掌握还不够，之后有机会更新吧。
