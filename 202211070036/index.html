helloOS的实现

注明：本系列实验基于彭东老师的操作系统实战课程，分享内容仅供作者本人留档阅读。

任意操作系统的实现，离不开引导程序的指引。由于本次实验仅仅是一次思路探索，做出的所谓OS也不过是一个单纯的黑屏和提示字，所以引导程序借用了成熟的GRUB引导程序，并出于虚拟机引导程序版本不同的考虑，在引导头引用上，使用了两个不同版本的必需代码。

本次虚拟机环境为Ubuntu20.0.8，带有图形化界面。值得一提的是，由于本次demo使用了GRUB，故不能在WSL上进行实际操作，因为WSL并不需要这样的引导头进行启动，故该虚拟机是vmware虚拟机。

除了针对引用GRUB部分进行的介绍外，还有一部分是重要的系统初始化。这些初始化代码主要进行了如下工作：首先是初始化了各个寄存器，随后令机器进入保护模式，最后引导CS:IP指向主函数。以上是汇编部分的整体情况。

MBT_HDR_FLAGS EQU 0x00010003
MBT_HDR_MAGIC EQU 0x1BADB002
MBT_HDR2_MAGIC EQU 0xE85250D6
global _start    ;    导出_start符号
extern main    ;    导入main函数符号
section .start.text;    定义.start.text节
bits 32    ;    汇编成32为代码
_start:
jmp _entry
ALIGN 8
mbt_hdr:
dd MBT_HDR_MAGIC
dd MBT_HDR_FLAGS
dd -(MBT_HDR_MAGIC+MBT_HDR_FLAGS)
dd mbt_hdr
dd _start
dd 0
dd 0
dd _entry

ALIGN 8
mbt2_hdr:
DD MBT_HDR_MAGIC
DD 0
DD mbt2_hdr_end - mbt2_hdr
DD -(MBT_HDR_MAGIC + 0 + (mbt2_hdr_end - mbt2_hdr))
DW 2, 0
DD 24
DD mbt2_hdr
DD _start
DD 0
DD 0
DW 3, 0
DD 12
DD _entry
DD 0
DW 0, 0
DD 8
mbt2_hdr_end:

ALIGN 8
_entry:
cli
in al, 0x70
or al, 0x80
out 0x70, al
lgdt [GDT_PTR]
jmp dword 0x8 : _32bits_mode
_32bits_mode:
mov ax, 0x10
mov ds, ax
mov ss, ax
mov es, ax
mov fs, ax
mov gs, ax
xor eax, eax
xor ebx, ebx
xor ecx, ecx
xor edx, edx
xor edi, edi
xor esi, esi
xor ebp, ebp
xor esp, esp

mov esp, 0x9000

call main

halt_step:
halt
jmp halt_step
GDT_START:
knull_dsc: dq 0
kcode_dsc: dq 0x00CF9E000000FFFF
;段基地址=0，段长度=0xfffff;G=1,D/B=1,L=0,AVL=0 ;P=1,DPL=0,S=1;T=1,C=1,R=1,A=0
kdate_dsc: dq 0x00CF92000000FFFF
k16cd_dsc: dq 0x00009E000000FFFF
k16da_dsc: dq 0x000092000000FFFF
GDT_END:
GDT_PTR:
GDTLEN dw GDT_END-GDT_START-1
GDTBASE dd GDT_START

随后是针对main函数，由于helloOS操作系统并没有C语言运行环境，所以printf()函数编译后在操作系统中是失效的——没有其他库中包含该函数！所以需要自己编写。

那么针对这个问题，解决方法就是研究“屏幕”和显示到底是个什么东西。显卡有两种工作模式，一种是字符模式，一种是图形模式。显然在这个demo中我们只需要关心字符模式即可。在显卡通用的VSEA标准下，显卡提供一个VGABIOS的固件程序，该固件程序运行在字符模式下时，把屏幕分成24×80的格间，每个格间映射一片内存——从0xb8000开始，每两个字节对应一个字符，第一字节是ASCLL码，第二字节是字符颜色。

显然，我们只需要将helloOS中printf的内容从0xb8000开始，每隔一个内存空间就映射一个字符即可，具体代码实现就不粘贴了。

helloOS的主要代码就告一段落了。剩下的代码就是有关Makefile的制作，lds文件的安排了，这些不属于主线内容，而且又是一个大坑，所以就暂且不表，放在之后。

随后便是实操上的困难。在我这个版本的ubuntu，开机是没有选择系统界面的，这需要去修改/etc/default/grub文件，将其hide属性注释或删除，timeout属性延长至20-30，GRUB_CMDLINE_LINUX_DEFAUL这个属性设置为text。

最后执行sudo update-grub该部分修改就结束了。

然后还要修改的文件是/boot/grub/grub.cfg，在其内部添加helloOS启动项，注意set root属性，需要通过指令df确定，而其余陌生属性通过上下文确定。启动项模板如下：

menuentry 'helloOS'{
    insmod part_msdos#上下文
    insmod ext2#上下文
    set root = 'hd0,msdos1'#数字按情况修改
    mutiboot2 = /boot/helloOS.bin#提前放好编译好的操作系统文件
    boot
}

随后重启操作系统，选择helloOS，这样我们的“自研”操作系统便横空出世了。
